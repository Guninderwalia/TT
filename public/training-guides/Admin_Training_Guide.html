TaskTango — Administrator Guide

How to run the system: employees, departments, holidays, payroll, performance, and audit trail. Updated for Production v3.5.

🆕 What's New (V10 → V3.5)

This release rolls up everything from the V10 / V11 hotfix line through to V3.5. Below is a quick reference — the rest of the guide describes how each area works in detail.

Dashboard Analytics

The admin dashboard now shows three live charts under Analytics, sitting between the stat cards and the Probation table:

  • Attendance — Last 30 Days: stacked area chart showing Present / Absent / Leave per day across the whole company.
  • Headcount by Department: doughnut chart breaking the active workforce by department.
  • Today's Attendance Status: doughnut showing the current-day split of Present / Absent / Leave / Half-day.

The 30-day chart is powered by a new aggregation handler so it doesn't slow the dashboard down even on larger databases.

Offboarding Flow (replaces Delete-as-deactivate)

Employees → row actions now has a 👋 Offboard button that records the departure properly instead of silently flipping a status flag:

  1. Click Offboard on the employee's row.
  2. Pick the Last Working Day, Exit Reason (Resignation / Termination / Contract End / Retirement), and optional notes.
  3. Tick off the HR exit checklist: Final salary processed · Security deposit refunded · Equipment returned · System access revoked.
  4. Confirm. The system flips the user to inactive, stamps employment_records.end_date, auto-cancels any pending leave requests starting after the last working day, and writes a full audit-log entry including the checklist.

Toggle 👥 Past Employees at the top of Employees to see the inactive list. Each past row has a ↩️ Reactivate action if you need to bring them back.

Performance Review — Cleaner Filters

  • The Last Increment column is gone — its data lives only inside the Edit Employee modal now.
  • New Employee dropdown sits after Department, so you can drill into a single person's KPIs instead of scanning the whole list.
  • The Department filter now actually works (the old version silently excluded everyone when a department was picked).

Chat Upgrade

The chat launcher moved to the dashboard header next to the 🔔 notification bell. It also gained:

  • Desktop notifications for incoming messages when the chat panel is closed or the window isn't focused.
  • Auto-jump straight to the most recent unread thread when you click 💬, instead of dumping you on a blank contact list.
  • Pulsing unread badge so you can't miss it.
  • Emoji picker (😊 in the composer) with ~80 popular emojis inserted at the cursor.
  • File attachments via the 📎 button — up to 10 MB. Images preview inline; other files open in the default OS app on click.

Mobile-Responsive

The full app now adapts to tablet (≤900 px) and phone (≤540 px) screen widths. The sidebar collapses to a top bar, data tables get horizontal scroll, modals run near full-width, and stat grids stack into a single column. Useful when you need to approve something from a phone.

Leave Rollover Policy (V2.0)

Each leave type now has its own carry-forward / encashment policy editable from Settings. The system writes a per-employee rollover log every 1 January so you have a HR-compliance trail of who carried what.

Getting Started

  1. Open TaskTango. Sign in with your administrator account (default username admin, password password; change it on first login).
  2. You'll land on the Administrator Dashboard with the full menu on the left.
  3. The very first time you use the system, work through Departments → Employees → Holidays in that order. Then everyone else can log in and the daily flow works.

The Menu, One Tab at a Time

🏠Dashboard

Your control panel — total employees, today's attendance summary, pending approvals, and any alerts.

What to do here: Glance at it once a day. If anything is red or showing pending approvals, click through to deal with it.

👥Employees

The master employee list. Everything about each person starts here.

To add a single employee:

  1. Click + Add Employee.
  2. Fill in name, email, phone, username, department, role, salary, start times, banking details, and joining date.
  3. Tick Team Lead if they manage that department.
  4. Save. The employee's initial password is password until they log in and change it.

To bulk-import:

  1. Click Download Template to get a blank Excel file with the right column headings.
  2. Fill in one row per employee in Excel.
  3. Click Upload Employees, choose your file. The system shows a preview.
  4. Confirm. All valid rows are created in one go.

To edit: Click the pencil icon next to an employee. Change anything and Save.

To reset a password: Click the key icon next to the employee. They'll be told to set a new password on next login.

To export everyone: Click Export Employees — you get an Excel file matching the import format.

Tip: Always assign a department and a role. The role drives what menu the employee sees when they log in.

🗂️Departments

Create and manage the company's departments (e.g. Evergrow, NCFS, Marketing, Accounting).

To create a department: Click + Add Department, give it a name and short description, save.

To assign a team lead: Open a department, choose an employee from the dropdown, save. That person now sees the Lead Dashboard when they log in.

Heads up: You can have only one lead per department at a time. Assigning a new one replaces the previous lead.

📅Attendance

The company-wide attendance grid. See who's present, absent, on leave, or marked half-day for any day or month.

To bulk-mark days as present:

  1. Select the date range and employees.
  2. Click Mark Selected as Present.
  3. Confirm. Useful for catching up after a system migration or training period.

To change one cell: Click the cell, choose a new status (Present / Absent / Leave / Half-day), save.

🎉Holidays

The public holiday list used across calendars (leave, attendance, payroll).

To add a holiday: Click + Add Holiday, set the date, name, and a short description, save. It immediately appears on all employees' calendars.

To edit or remove: Use the row-level Edit/Delete buttons.

When to add holidays: Beginning of the year, plus any time the government declares an extra day off. Days marked as public holidays do not count against attendance or leave balance.

✅Leave Approvals

The admin's leave queue — requests that have already been approved by a department lead and are waiting for your final sign-off.

How to handle a request:

  1. Read the employee, dates, days, reason, and the lead's notes.
  2. Click Approve for final approval — this deducts the employee's leave balance and auto-marks those days as Leave in Attendance.
  3. Click Reject if needed; you'll be prompted for a reason. The employee sees this.
Two-stage approval flow: Lead approves first → admin approves second. Balance is only deducted after your approval. Requests from leads themselves come straight to you, skipping stage one.

⏰Time Logging

A read-only view of every employee's time logs, by month.

What to do here: Spot people who haven't been logging hours, or who have unusually long or short days. Use it as a sanity check before running payroll.

📊Performance Review

Company-wide performance scores. Each employee has an Overall Score out of 100, broken into six metrics.

The six metrics and their weights:

⭐ Skills40%
👤 Manager Rating20%
📅 Attendance10%
⏰ Punctuality10%
📊 Consistency10%
🚦 Lateness10%

Colour badges: Green ≥ 80 Yellow 60–79 Red < 60

What to do here:

  • Filter by department or date range.
  • Click Review next to anyone to add a star rating (1–5) and written comments — feeds the Manager Rating.
  • Click Skills to rate them on individual skills (1–5) — feeds the Skills score.
See the separate Performance Review Scoring Guide for the full formula explained in plain English.

💷Payroll

Run monthly payroll for the whole company.

To process a month:

  1. Pick the payroll month and year.
  2. Click Process Payroll. The system calculates each employee's base salary, overtime, bonuses, attendance deductions, probation deposit deductions, and net pay.
  3. Review the table — make sure everyone looks right.
  4. Add ad-hoc expenses or bonuses where needed.
  5. Mark the run as Processed when payments are released.

Payslips: Once processed, each employee can view their payslip on their own dashboard.

🔒Security Deposits

Probation-period security deposits for new joiners.

To set up a deposit: Click + Add Deposit, choose the employee, total deposit amount, and the months it should be deducted across (e.g. Jan–Feb of next year). Save.

What happens: The system automatically deducts a portion each payroll month until the full amount is held. Once probation is completed, you can mark the deposit as Released to refund.

🛡️Audit Logs

A complete history of every change made in the system.

What you see: Who did what, when, and the before/after values. Use filters to narrow by user, entity type (Employee / Leave / Payroll / etc.), or date range.

Use this when: Someone asks "who changed X?" or "when was this updated?" — you'll find the answer here.

First-Time Setup Checklist

  1. Change the default admin password. Bottom-left → Password → set a strong new one.
  2. Create departments (Departments tab).
  3. Add a Managing Director or Administrator account if needed (Employees → Add Employee, set role to MD or Administrator).
  4. Add this year's public holidays (Holidays tab).
  5. Bulk-import existing employees using the Excel template (Employees → Download Template → Upload Employees).
  6. Assign team leads for each department (Departments → choose lead).
  7. Let everyone log in. They'll be prompted to change their password on first login.

Important Safety Notes

Deleting an employee: This deactivates them — their historical data (payslips, attendance, leave) is preserved. Don't try to "remove" people by editing their records to look blank; just use the Delete button.
Bulk import overwrites? No — bulk import only creates new employees. If someone's email already exists, that row is skipped with an "Email already exists" note.
Resetting passwords: Once you reset someone's password, the old one stops working immediately. Make sure to tell the employee the temporary password is password (they'll change it on next login).

Frequently Asked Questions

An employee tells me they can't approve leave even though they're a Lead.

Check the Department they belong to and make sure they're set as "Team Lead" for that department. Open the employee in the Employees tab and look at the Team Lead checkbox; also confirm in Departments that they're the assigned lead.

Where is the Annual Leave entitlement set?

The default is 25 days per year, prorated by joining date. If you need a different default for new joiners, ask your IT team — it's controlled at install time.

An employee's performance score is showing zero even though they have time logs.

The most likely reason is they have no Manager Review and no Skill assessments yet — together those are 60% of the score. Give them a review and rate their skills via Performance Review → their row → Review / Skills buttons.

Can I run payroll for a single employee?

Payroll runs for the whole company at once, but you can edit a specific employee's row before marking the run as Processed.

Where do I see who deleted what?

Audit Logs. Filter by Action = DELETE or by the affected employee's ID.

An employee's leave balance looks wrong.

Open Leave Approvals or Audit Logs and check what's been approved this year. If the balance still looks off, re-opening the Leave Requests page for that user re-seeds their balance row from the current leave types.
