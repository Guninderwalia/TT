1. What "Admin" means in TaskTango

An Administrator (also called MD, Manager, or Administrator depending on the role label) has full control of every record in the system. There is no higher role.

As an admin you can:

  • Create, edit, offboard, reactivate any employee
  • Create departments and assign team leads
  • Approve / reject leave (the final stage in the approval chain)
  • Edit anyone's time logs (the only role that can)
  • Write performance reviews for anyone
  • Upload HR documents for any employee
  • Configure leave types, holidays, settings, timezone
  • View & export the audit log
  • Take database backups, restore, and wipe test data
Because the admin role has root powers, treat your password like a vault key. Use a unique, long password and don't reuse it elsewhere.

2. First-time setup checklist

If TaskTango was just installed, do these in order:

  1. Sign in with the bootstrap admin credentials your installer provided.
  2. Change the bootstrap password immediately. (Settings → Account → Change Password)
  3. Set the office timezone. Settings → General → Office Timezone. Default is Asia/Kolkata. Pick yours.
  4. Define your departments. Employee Manager → Departments → + New.
  5. Create leave types & allocations. Settings → Leave Types. At minimum: Annual, Sick, Casual.
  6. Add public holidays. Settings → Holidays. Bulk-paste if you have them in a list.
  7. Add the first few employees. Employee Manager → + New Employee.
  8. Assign department leads. Edit a department → set Lead.
  9. Send each new employee their temporary password securely (e.g., 1Password share link, not email plaintext).
  10. Take a baseline backup. Settings → Backup → Download.
This 10-step checklist gets you to a working system in ~30 minutes for a 20-person company. Bigger companies usually use the Excel bulk import (Section 5).

3. The admin dashboard — every widget explained

📊 Headcount by Department (bar)

How many employees in each department. Click a bar to filter the Employee Manager to that department.

🟢 Today's Attendance Status (donut)

Company-wide breakdown of Present / Late / Absent / On Leave. Refreshes every 30 seconds.

📈 30-day Attendance Trend (line)

Stacked daily attendance for the past 30 days. Useful for spotting attendance drift after holidays.

👥 Live Company Status

Same idea as the lead's Team Status, but across the whole company. Counts of Working / On Break / Idle / Signed Off / Absent / On Leave, plus the full list with presence dots.

📅 Upcoming Leaves (company-wide)

Every approved leave starting in the next 30 days. 📅 Export to Calendar button gives an .ics file — drop it in Outlook/Google to subscribe.

🔔 Pending Approvals counter

Quick badge showing how many leave requests are waiting for your final sign-off. Click to jump to the Leave Approvals queue.

4. Employee Manager — full CRUD walkthrough

Creating a new employee

  1. Sidebar → Employee Manager → + New Employee.
  2. Fill the form:
    • Full Name — exactly as on payroll documents
    • Email — used for login; must be unique
    • Phone
    • Role — Administrator / Manager / MD / Lead / User. (Most hires are "User".)
    • Department
    • Is Department Lead? — tick if they should approve their team's leave
    • Employment Type — Full-time / Part-time / Contractor
    • Joining Date — used to calculate tenure
    • Probation End Date — see Cron section 15 for what happens at this date
    • Base Salary — only visible inside this form, never on the profile
    • Bank Account & IFSC — for payroll exports
    • Temporary Password — auto-generated if you leave blank
  3. Click Save.
  4. Send the temporary password to the employee securely. They'll be forced to change it on first login.

Editing an employee

Click their row, then Edit. Every field is editable except the email (the unique-ID anchor). To change someone's email, contact support — or in a pinch, offboard and recreate.

Deleting vs offboarding

You almost never want to delete — that wipes all their history. Use Offboard instead (Section 6).

HR onboards three engineers on the same day. Sneha creates each in Employee Manager, sets department to Engineering, ticks "User" role, leaves the password field blank to auto-generate. Each generated password is copied into a 1Password vault entry the new joiner has access to.

5. Bulk import & export (Excel)

Exporting to Excel

  1. Employee Manager → ⬇ Export to Excel.
  2. A file downloads: tasktango-employees-YYYY-MM-DD.xlsx.
  3. Open in Excel. Columns: ID, Full Name, Email, Phone, Department, Role, Joining Date, Status, etc.

Importing from Excel

  1. Download the template first (Employee Manager → ↗ Import from Excel → Download Template).
  2. Fill in rows. Required columns: Full Name, Email, Role, Department.
  3. Save as .xlsx.
  4. Click ↗ Import again and select your file.
  5. The preview screen shows how many rows are new, duplicate email (will skip), invalid (will skip).
  6. Click Confirm Import. Each new employee is created with an auto-generated temporary password.
Run an export before any large import — that gives you a snapshot to compare against if anything goes wrong.
The importer never overwrites existing employees. If you need to bulk-update existing rows, do it via the spreadsheet then re-enter each manually — there is no bulk-edit yet.

6. Offboarding & reactivation

Offboarding an employee

  1. Employee Manager → click the row → Offboard.
  2. Fill the modal:
    • Last working day — date the account should stop working
    • Reason — Resignation / Termination / End of contract / Other
    • Notes — internal HR memo (not visible to other employees)
  3. Click Confirm Offboard.

What offboarding actually does

  • Account is marked inactive — the employee can no longer log in
  • They stop appearing in the chat contacts list, presence widgets, org chart
  • Their historical records (time logs, leaves, reviews) are preserved for compliance
  • Outstanding leave requests are auto-cancelled

Reactivating someone (boomerang hire)

Employee Manager → Filter → Show Offboarded → click the row → Reactivate. The account turns back on with the original email but a fresh temporary password.

Vikram resigns on 30 April with last day 31 May. On 25 May admin offboards him with reason "Resignation". Vikram continues to log in normally until 31 May, then his account is auto-disabled on 1 June. His past time logs and reviews remain visible to admin for the statutory 7-year retention.

7. Departments & team leads

  • Create: Employee Manager → Departments → + New Department. Name + description.
  • Assign a lead: Edit a department → Lead dropdown → pick from existing employees.
  • The selected employee automatically gets is_department_lead = 1 and now sees the Lead Dashboard.
  • Move someone between departments: Edit the employee → change Department field → Save. Their pending leave requests re-route to the new lead.
A department without a lead means all leave requests from that team skip straight to the admin queue. If you don't want that load, always assign a lead.

8. Leave types & balances

Configuring leave types

Settings → Leave Types. Each leave type has:

  • Name (Annual, Sick, Casual, Bereavement, Maternity, etc.)
  • Default annual allocation (days per year)
  • Carry-over policy (carry unused days into next year? cap?)
  • Requires reason? — toggle
  • Requires approval? — defaults to yes

Custom balance per employee

Edit any employee → Leave Balances tab → adjust their allocation for a specific type. Use this for senior staff with negotiated extras, or new joiners on pro-rata.

Year-end roll-over

On 1 January each year TaskTango auto-allocates the default balance to every active employee, subject to carry-over rules. You can preview this rollover in Settings → Leave Year-End Preview before it runs.

9. Leave approvals (single & bulk)

Where the queue lives

Sidebar → Leave Approvals. Shows requests that have already passed the lead stage (or are from a leaderless department) and need your final sign-off.

Single approve / reject

  1. Click Approve — days are deducted from the employee's balance, attendance calendar marks those days as On Leave, employee gets notified.
  2. Or Reject — modal asks for a reason. Employee gets notified with your reason.

Bulk approve / reject (v4.5)

  1. Tick checkboxes on the left of each row, or use the header checkbox to select all.
  2. A green action bar appears: ✓ Approve Selected / ✗ Reject Selected.
  3. Bulk approve runs all approvals in parallel; you get one toast: "Approved 12 requests."
  4. Bulk reject opens one reason modal — the same reason is sent to every selected employee.
After Diwali 27 requests pile up. Admin opens Leave Approvals, sees 22 are clean and pre-approved by leads. Ticks the header checkbox, ticks off the 5 contentious ones, hits Approve Selected. Done in 10 seconds. Handles the 5 individually with notes.

10. Time logs — corrections & reporting

Viewing

Sidebar → Time Logging. Filter by employee, date range, status. Export the visible rows to PDF.

Editing & deleting (admin-only)

As admin you see Edit and Delete buttons on every row in the History tab — these are hidden for leads and employees by design.

When to correct a log

  • Employee forgot to sign out — fix the sign-out time
  • Employee was wrongly auto-marked Absent — delete the absent record
  • Break time entered by mistake — delete the break
Every edit is recorded in the Audit Log with your name, the old value, the new value, and the timestamp. Make sure your reason is defensible.

11. Performance review module

Sidebar → Performance Reviews. As admin you can write a review for anyone, including department leads.

Writing a review

  1. Click + New Review next to the employee.
  2. Set the review period (date-from / date-to).
  3. Score each category 1–5 (see the separate Performance Review Scoring Guide).
  4. Strengths, Areas to improve, Overall comments.
  5. Save.

The PDF output

v4.5 PDF improvements:

  • Stars now render as the readable text 4.2 / 5 instead of missing-glyph boxes
  • Employee row shows Team Lead name (was Employee ID before)
  • Signature block shows "Reviewed by Team Lead: Name"

12. Document management (HR uploads)

Sidebar → Employee Manager → click an employee → Documents tab.

  • Upload: drag a file in, or click + and pick from disk. Types: offer letter, contract, ID copy, address proof, NDA, certificates.
  • Replace: upload again with the same type — the previous version is kept in audit but the latest is shown.
  • Delete: click the trash icon. Audit-logged.
  • What the employee sees: they can download their own documents, but never upload or delete.

13. Org Chart maintenance

The Org Chart is auto-generated from your data. You don't draw it manually — it builds itself from:

  1. The MD / Admin at the top (any user with the MD role label)
  2. Department Leads on the next level
  3. Team members underneath their lead

To change the chart, change the underlying data: assign a different lead to a department, move someone to a new department, etc.

Sharing the chart

Open Org Chart → ⬇ Export to PDF. Output is A3 landscape with photos. Use it in board packs, new-joiner welcome packs, or the office wall.

14. Chat, presence & calls

You use chat the same way every other user does. Admin-specific notes:

  • You see everyone in the contacts list, not just your department.
  • The presence dots in chat are computed live from the same data the dashboards use.
  • You can call any user 1:1 (audio / video). Group calls are not in v4.5.
  • Chat attachments are stored in the database; large files (>5 MB) bloat backups — set a sensible policy with your team.

15. Background cron jobs

TaskTango runs an internal scheduler that triggers once an hour. Three jobs run inside it:

JobWhen it firesWhat it does
Auto-mark Absent 30 min after each employee's scheduled start time If they haven't signed in, marks them Absent for today. They can still sign in later but the Absent flag stays unless admin clears it.
Probation auto-flip Any hour where someone's probation_end_date has passed Sets is_probation = 0. The status pill on their profile flips from 🔴 Probation to 🟢 Permanent.
Auto Sign-out 23:00 office time Anyone still showing as signed in is automatically signed out using their employment record's end_time. Prevents 24-hour-day records.
Cron events also appear in the Audit Log with the actor "system" so you can see exactly what fired and when.

16. Audit Log — filter, CSV, purge

Sidebar → Audit Log & System Security. Every meaningful action lands here.

What's logged

  • Logins / logouts (with IP address)
  • Password changes (no plaintext stored)
  • Employee create / update / delete
  • Leave approve / reject (at each stage)
  • Time-log edits & deletes
  • Document uploads & deletes
  • Cron job runs

Filtering & export

  1. Use the filter row at the top: free-text search, Action, Entity type, User, Date range.
  2. The stat cards above show: Total · Filtered · Last 24h · Most-common action.
  3. Click any row to expand the before/after diff.
  4. Click ⬇ Export CSV to download the currently-filtered rows.

Clearing logs

  • 🗑 Clear Filtered — deletes only the rows matching your current filter
  • ⚠ Clear All — deletes every audit log row in the database
Clearing audit logs is irreversible and may breach your data retention policy. Take a database backup (Section 17) before any purge, and only purge to free disk space, never to "tidy up".

17. Settings — timezone, backup, security

🌐 Office timezone

Default Asia/Kolkata. Affects every "today" calculation — sign-in, attendance, cron jobs. Change once at setup; avoid changing later without warning the team.

💾 Database backup

Click Download Backup → a single .sqlite file downloads. Store it in cloud storage with versioning. Take backups weekly at minimum, daily for active companies.

♻ Restore from backup

Upload a previously-downloaded .sqlite. This replaces the entire database — current data is wiped. Used for disaster recovery; never as a "go back" button without warning.

🔐 Force password reset

Bulk-resets every user's password to a system-generated temp. Use only after a suspected breach.

📅 Holidays

Add public holidays one at a time, or bulk-paste from a list. Holidays show in attendance as Holiday and don't count toward leave deductions.

18. Wipe Test Data (use with care)

Settings → Wipe Test Data panel. Designed for clearing out demo / pilot data before going live.

What gets wiped

  • All employees except admins / MDs
  • All time logs, attendance records, leave requests
  • All performance reviews, documents, chat messages
  • All audit log entries

What survives

  • Admin / MD accounts
  • Role definitions
  • Leave types
  • Settings (timezone, holidays)

How to use it safely

  1. Take a backup first (Settings → Download Backup). Always.
  2. Open the Wipe Test Data panel.
  3. Type WIPE in the confirmation box (case-sensitive — you cannot type-and-forget your way through this).
  4. Click Wipe Now.
  5. Sign back in — you'll see a clean slate.
Wipe Test Data is irreversible without your backup. Never run it on a production database without confirming the backup file opens correctly first.

20. Troubleshooting playbook

Q: Someone can't log in

1) Check Caps Lock. 2) Reset their password — Employee Manager → row → Reset Password → copy temp → send securely. 3) Confirm their account isn't offboarded — Show Offboarded filter.

Q: An employee's leave balance looks wrong

Edit the employee → Leave Balances tab → manually set the correct number. Save. Audit-logged.

Q: An entire day shows everyone as Absent

The cron auto-marked them because the office timezone was wrong (start time computed wrong). Fix the timezone in Settings, then delete the bogus absent records from each employee's timesheet.

Q: The Org Chart has a "ghost" employee

Probably an offboarded user that didn't get re-rendered. Refresh the page; if still there, edit their record and confirm their is_active flag is 0.

Q: PDF reports show boxes instead of letters

You're on a pre-v4.5 build. Upgrade — v4.5 uses ASCII-safe stars and renders cleanly.

Q: I deployed to a server but mic/camera permission keeps failing

HTTPS is required for getUserMedia in browsers. If you're on plain HTTP, the API returns undefined. Put TaskTango behind a TLS proxy (Fly.io does this automatically).

Q: Database file is huge

Largest contributors: chat attachments and old time logs. Use the Audit Log clear-filtered to remove logs older than N years; consider deleting closed chat conversations from the chat panel.

Q: Where is the database file on disk?

Desktop: %APPDATA%\TaskTango\tasktango.db on Windows, ~/Library/Application Support/TaskTango on macOS.
Web/Server: at USER_DATA_PATH from the env (Fly default: /data/tasktango.db).

Q: How do I restore a deleted employee?

If you used Delete, the only way back is to restore from a backup. If you used Offboard, just hit Reactivate. Always prefer Offboard.

Q: A team lead complains the bulk approve missed one

Possible the lead-approved-by step failed (network blip). Look at the Audit Log filtered by Action = LEAVE_APPROVE — you'll see which ones logged. The missing one needs a manual click.
